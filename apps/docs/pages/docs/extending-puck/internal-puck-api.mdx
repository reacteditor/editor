# Internal Puck API

Puck exposes its internal API as [`PuckApi`](/docs/api-reference/editor-api) for extending Puck with custom functionality within [custom fields](/docs/extending-editor/custom-fields), [compositional interfaces](/docs/extending-editor/composition) or [UI overrides](/docs/extending-editor/ui-overrides).

## Accessing the internal API

You can access [`PuckApi`](/docs/api-reference/editor-api) via two hooks:

- [`usePuck`](/docs/api-reference/functions/use-editor) - returns `PuckApi` as part of your component render lifecycle
- [`useGetPuck`](/docs/api-reference/functions/use-get-editor) - returns a function to access the latest `PuckApi` at call time

### Within the render lifecycle

To access the API within your render lifecycle, use the `usePuck` hook. You can use a selector to limit re-rendering to a specific part of the API.

```tsx copy
import { createUsePuck } from "@editoreditor/core";

const usePuck = createUsePuck();

const Example = () => {
  // Use a selector so you only re-render when the selected type changes
  const type = usePuck((s) => s.selectedItem?.type || "Nothing");

  return <h2>{type} selected</h2>;
};
```

See the [`usePuck`](/docs/api-reference/functions/use-editor) docs for a full API reference.

### Outside of the render lifecycle

Often it's not necessary to re-render your component when the PuckApi changes. Puck provides the `useGetPuck` hook for accessing the latest `PuckApi` at call time.

```tsx copy
import { useGetPuck } from "@editoreditor/core";

const Example = () => {
  const getPuck = useGetPuck();

  const handleClick = useCallback(() => {
    // Get the latest PuckApi value
    const { appState } = getPuck();

    console.log(appState);
  }, [getPuck]);

  return <button onClick={handleClick}>Click me</button>;
};
```

See the [`useGetPuck`](/docs/api-reference/functions/use-get-editor) docs for a full API reference.

## Usage in practice

Generally, you'll want to combine this with composition, UI overrides or custom fields.

Here's an example using the internal API to log the page data as JSON while retaining the standard Puck UI:

```tsx
import { Puck, createUsePuck } from "@editoreditor/core";

const usePuck = createUsePuck();

const JSONLogger = ({ children }) => {
  const appState = usePuck((s) => s.appState);

  useEffect(() => {
    console.log(appState);
  }, [appState]);

  return <>{children}</>;
};

export function Editor() {
  return (
    // Render the Puck context
    <Puck>
      <JSONLogger>
        {/* Since we're overriding Puck's children, we restore the standard Puck UI */}
        <Puck.Layout />
      </JSONLogger>
    </Puck>
  );
}
```
