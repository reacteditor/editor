# Internal Frontend API

Frontend exposes its internal API as [`EditorApi`](/docs/api-reference/editor-api) for extending Frontend with custom functionality within [custom fields](/docs/extending-editor/custom-fields), [compositional interfaces](/docs/extending-editor/composition) or [UI overrides](/docs/extending-editor/ui-overrides).

## Accessing the internal API

You can access [`EditorApi`](/docs/api-reference/editor-api) via two hooks:

- [`useEditor`](/docs/api-reference/functions/use-editor) - returns `EditorApi` as part of your component render lifecycle
- [`useGetEditor`](/docs/api-reference/functions/use-get-editor) - returns a function to access the latest `EditorApi` at call time

### Within the render lifecycle

To access the API within your render lifecycle, use the `useEditor` hook. You can use a selector to limit re-rendering to a specific part of the API.

```tsx copy
import { createUseEditor } from "@frontend/core";

const useEditor = createUseEditor();

const Example = () => {
  // Use a selector so you only re-render when the selected type changes
  const type = useEditor((s) => s.selectedItem?.type || "Nothing");

  return <h2>{type} selected</h2>;
};
```

See the [`useEditor`](/docs/api-reference/functions/use-editor) docs for a full API reference.

### Outside of the render lifecycle

Often it's not necessary to re-render your component when the EditorApi changes. Frontend provides the `useGetEditor` hook for accessing the latest `EditorApi` at call time.

```tsx copy
import { useGetEditor } from "@frontend/core";

const Example = () => {
  const getFrontend = useGetEditor();

  const handleClick = useCallback(() => {
    // Get the latest EditorApi value
    const { appState } = getFrontend();

    console.log(appState);
  }, [getFrontend]);

  return <button onClick={handleClick}>Click me</button>;
};
```

See the [`useGetEditor`](/docs/api-reference/functions/use-get-editor) docs for a full API reference.

## Usage in practice

Generally, you'll want to combine this with composition, UI overrides or custom fields.

Here's an example using the internal API to log the page data as JSON while retaining the standard Frontend UI:

```tsx
import { Editor, createUseEditor } from "@frontend/core";

const useEditor = createUseEditor();

const JSONLogger = ({ children }) => {
  const appState = useEditor((s) => s.appState);

  useEffect(() => {
    console.log(appState);
  }, [appState]);

  return <>{children}</>;
};

export function Editor() {
  return (
    // Render the Frontend context
    <Editor>
      <JSONLogger>
        {/* Since we're overriding Frontend's children, we restore the standard Frontend UI */}
        <Editor.Layout />
      </JSONLogger>
    </Frontend>
  );
}
```
